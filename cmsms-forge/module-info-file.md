## Module Info File (moduleinfo.ini)

The `moduleinfo.ini` file contains metadata about your module. It is auto-generated by CMSMS when you create an XML package, but you can also maintain it manually. The Module Manager reads this file to display module information.

### Format

```
[module]
name = "Holidays"
version = "1.0.0"
author = "Your Name"
authoremail = "you@example.com"
mincmsversion = "2.2.1"
lazyloadadmin = 0
lazyloadfrontend = 0
```

### Fields

| Field | Description |
| --- | --- |
| `name` | The module name (must match the directory and class name) |
| `version` | The current version number |
| `author` | The module author's name |
| `authoremail` | The author's email address |
| `mincmsversion` | Minimum CMSMS version required |
| `lazyloadadmin` | Whether the module supports lazy loading on admin requests (0 or 1) |
| `lazyloadfrontend` | Whether the module supports lazy loading on frontend requests (0 or 1) |

> **Note:** This file is auto-generated by CMSMS based on the return values of your module class methods (GetName, GetVersion, GetAuthor, etc.). You generally do not need to create or edit it manually — it is regenerated each time an XML package is built.
